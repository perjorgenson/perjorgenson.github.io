Eggboard: Collaborative Whiteboard for RPGs

February 5, 2026

Overview

I created a collaborative whiteboarding web app to help organize the direction of table-top roleplaying games I play with my freinds. The free tool we were using, Zoom’s in-built whiteboard, had some feature limitations in the free version, and looked kind of ugly. So I made something myself (with lots of help from Claude).

So what is it?

It’s a web-app, that you can open up on your personal computer. The scope of it is kept pretty small–I just want to be able to organize short and long text, notes, and images (eventually).

Users of Eggboard mainly live within a “board”, which is a bulletin-board-like environment where media elements can be created on the screen. There is the ability to zoom in/out, and move around elements on the board.

Collaboration is a core part of the app–unique users that have logged in can see other users’ presence, through their mouse location on the canvas as well as a list of logged-in users to the side. Edits made to content on the board are immediately reflected to all connected users.

The boards and content within them persist on the server-side. This allows long-term usage of the app for extended campaigns. At this point in time there are two types of content included in eggboard

  1. “Notes”, which are single-line strings of text.
  2. “Documents”, which are long-format files that are assumed to follow markdown formatting convention

Notes and Documents should handle 98% of the needs of what I want to use this tool for–simply organizing and sharing information from a narrative-driven game over weeks, months, or even years. Additional media types that I’d like to embed at some point are

  • Images
  • Video
  • Audio

How does it work?

Frontend:

  • React + Typescript
  • Using Konva for the canvas capabilities

Backend:

  • FastAPI + Python
  • Persistence handled through boring old files–markdown, json

Hosting:

  • Self-hosting for now, since it’s only to be used by myself and a few friends
  • Hardware is a linux mini-pc server that I have laying around, and doing some port forwarding on my home router

Tour of how to use

  1. Enter the IP address of the server. You’ll be prompted for a user name and an avatar
  2. Once you’ve entered that, you’ll be presented with a screen that lets create a new Board, or to open up an existing one. Since it’s probably your first time, click “Create New Board” and enter the name of it
  3. You will be sent into the board you just created, and see an empty board. Notable things on the screen, starting in the top left
    1. “Back to Boards” button: This brings you back to the previous screen, where you can create a new board or enter an existing one
    2. Board ID: the name of the board you are in
    3. Connection status–this will show is something has gone wrong. Green means good.
    4. “Create Note”: Click this to create a short, 1-line note that can be placed and interacted with on the board.
    5. “Create Document”: Click this to open up a dialog for creating a new long-form text document. You can create the title, add tags, and a description of the document.
    6. “browse documents”: View any document that currently exists, even if it is not currently in the active board.
    7. “Reset View”: Go back to the starting view of the board, in case you zoomed in too far or got lost
    8. Zoom indicator, and instructions for how to pan around
    9. Online Users Indicator: Lists online users, including yourself. To test out multiple users, open up a new incognito tab and enter the same board to see the collaborative features at work
  4. Media editing and interactions:
    1. Double left click on Notes or Documents to edit them
    2. Right click to bring up color options and more
    3. Single left click and drag to move documents around
    4. Delete key can be used to quickly delete something
    5. Click outside of the editor window to leave the editor
    6. Or press escape to leave the editor view
    7. If you delete it, it’s gone forever! (TODO make things undo-able)
